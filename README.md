## 요구사항 정의

1. 주문 데이터를 저장하여야 한다.
2. 재고관리를 하여야 한다.
3. 포인트를 사용해야 한다.
4. 주문, 재고, 포인트 데이터의 정합성이 맞아야 한다.
5. 동일한 주문은 1번만 이루어져야 한다.

## 요구사항 충족 현황 (monolithic)

| # | 요구사항          |           상태            |
|---|---------------|:-----------------------:|
| 1 | 주문 데이터 저장     |            ✅            |
| 2 | 재고관리          |            ✅            |
| 3 | 포인트 사용        | ⚠️ `userId` 하드코딩(학습 목적) |
| 4 | 주문·재고·포인트 정합성 |        ✅ 단일 트랜잭션        |
| 5 | 동일 주문 1회 보장   |      ⚠️ 분산락 TTL 부재      |
